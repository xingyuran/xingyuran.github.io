旋转数组的最小数字

题目描述

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。 输入一个非递减排序的数组的一个旋转，输出旋转数组的最小元素。 例如数组{3,4,5,1,2}为{1,2,3,4,5}的一个旋转，该数组的最小值为1。 NOTE：给出的所有元素都大于0，若数组大小为0，请返回0。

思路

有三种情况

  1. rotateArray [mid] > rotateArray [begin]
    最小的数肯定在rotateArray [mid] 的右边。
  2. rotateArray [mid] < rotateArray [begin]
    最小的数肯定在rotateArray [mid] 的左边。
  3. rotateArray [mid] == rotateArray [begin]
    这种情况不好判断，如 [1,0,1,1,1]，[1,1,1,0,1]，只有 begin++ 挨个判断

知识点

  • STL 中容器是否为空的判断条件 v.empty() 和 v.size()==0 的区别：对于std::vector来说，其实效率没区别，但是对于std::list来说(gcc 4.6-)，empty()的效率是O(1)，size()的效率是O(n)。
    C++11对size()的效率做了一致性的规定。所以在gcc 4.7+, std::list的复杂度也是O(1)了。原因是增加了一个“size”的变量来存长度。

用两个栈实现队列

题目描述

用两个栈来实现一个队列，完成队列的Push和Pop操作。 队列中的元素为int类型。

思路

入队：将元素进栈A
出队：判断栈B是否为空，如果为空，则将栈A中所有元素pop，并push进栈B，栈B出栈；
如果不为空，栈B直接出栈。
补充：(用两个队列实现栈的功能)
入栈：将元素进队列A
出栈：判断队列A中元素的个数是否为1，如果等于1，则出队列，否则将队列A中的元素 以此出队列并放入队列B，直到队列A中的元素留下一个，然后队列A出队列，再把 队列B中的元素出队列以此放入队列A中。

知识点

  • std::stack::empty
    true if the underlying container’s size is 0, false otherwise.
  • stack::pop 没有返回值

内部网关协议OSPF

OSPF协议的基本特点

这个协议的名字是开放最短路径优先 OSPF (Open Shortest Path First)。 它是为了克服 RIP 的缺点在 1989 年开发出来的。
“开放”表明 OSPF 协议不是受某一家厂商控制,而是公开发表的.使用了 Dijkstra 提出的最短路径算法 SPF。
OSPF 只是一个协议名字,并不表示其他的路由选择协议不是”最短路径优先”。事实上，所有的自制系统内部使用的路由协议（包括RIP协议）都是要寻找一条最短路径。

OSPF 最主要的特征就是使用分布式的链路状态协议，而不是像 RIP 的距离向量协议，OSPF的三个要点和 RIP 都不一样：

  1. 向本自制系统中所有路由器发送信息。这里使用的是泛洪法（flooding）。
  2. 发送的信息就是与本路由器相邻的所有路由器的链路状态，但这只是路由器所知道的部分信息。
  3. 只有当路由信息发生变化时才向所有路由器用泛洪法发送信息。

OSPF 的更新过程收敛得很快尤其重点。

为了使 OSPF 能用于规模很大的网络，OSPF将一个自治区再划分为如干戈更小的范围，叫区域。

主干区域的标志符规定为 0.0.0.0，是用来连接其他在下层的区域。从其他区域来的信息都由区域边界路由器进行概括。主干区域的路由器叫主干路由器。

OSPF 不使用 UDP 而**直接用 IP 数据报传送。

OSPF的五种分组类型

  1. 问候分组;
  2. 数据库描述分组;
  3. 链路状态请求分组;
  4. 链路状态更新分组;
  5. 链路状态确认分组;

OSPF 规定，每两个相邻路由器每隔十秒要交换一次问候分组。这样就可以确认哪些路由器是可达的。

泛洪法（如上图）转发收到的分组时，要将其上游路由器除外，泛洪法在收到更新分组后要发送确认（收到重复的更新分组只需确认一次）。图中绿色箭头表示确认分组。

装载问题

问题描述

有一批集装箱要装上 2 艘重量分别为 c1，c2 的轮船，其中集装箱 i 的重量为wi，且w1+w2+…+wi<=c1+c2。
装载问题要求确认，是否有一个合理的装载方案可以将这n个集装箱装上这 2 艘轮船。如果有，找出一种装载方案。

算法设计

容易证明，如果一个给定的装载问题有解，则用下面的策略可以得到最优装载方案。

  1. 首先将第一艘轮船尽可能填满;
  2. 然后将剩余的集装箱装上第二艘船。

将第一艘船尽可能装满等价于选取全体集装箱的一个子集, 使得盖子集中集装箱重量之和最接近 c1, 在这样的情况下, 装载问题等价于 0-1 背包问题.

重建二叉树

题目描述

输入某二叉树的前序遍历和中序遍历的结果，请重建出该二叉树。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。例如输入前序遍历序列{1,2,4,7,3,5,6,8}和中序遍历序列{4,7,2,1,5,3,8,6}，则重建二叉树并返回。

思路

前序序列必定是一个个顺序插入，中序序列中一个节点的左边是它的左分支，右边是它的右分支。根据这个特点做递归实现。

知识点

  • 中序序列中任一个节点的左边是它的左分支，右边是它的右分支。
  • 后序序列最后一个节点是该树的根节点。

替换空格

题目描述

请实现一个函数，将一个字符串中的空格替换成“%20”。例如，当字符串为We Are Happy.则经过替换之后的字符串为We%20Are%20Happy。

思路

因为给的 str 数组的空间大小是固定的，所以先判断将空格替换为 %20 后，str 的空间是否能装下，若不能变返回错误。在插入的时候要从后往前插入，若是从前往后插入，当遇见空格时，就要向后移动两次，导致移动次数过多。

知识点

  • strlen 函数返回的是字符串长度，sizeof 运算符返回的是类型(变量)的长度。
  • 当适用一静态地空间数组， sizeof 归还全部数组的尺寸，不能返回动态地被分配了的数组或外部数组大小。
  • 数组做sizeof的参数不退化，传递给strlen就退化为指针了。
  • sizeof可以用类型做参数，strlen只能用char*做参数，且必须是以’’\0’’结尾的。sizeof还可以用函数做参数，非类型参数可以不加括弧。

题目描述

思路

因为数组是有序的，所以为了减少查找次数，从数组的左下方也就是array[row-1][0]开始找，若是target比该数大，就找该列的上一行，若是target比该数小就找该行的下一列。
