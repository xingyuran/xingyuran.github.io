链表中倒数第k个节点

题目描述

给出一个链表，输出倒数第 k 个节点。

思路

两个指针，同时指向头节点，第一个指针先向后移 k 个节点，然后两个指针同时向后移动，直到第一个指针 next 为空，第二个指针指向的就是所求。注意考虑代码的鲁棒性，当 k<=0 时的情况，链表为空的情况，链表长度不足 k 的情况。

知识点

  • 代码的鲁棒性（考点）
  • 输出链表的中间节点（延伸）

调整数组顺序使奇数位于偶数前面

题目描述

思路

  • 从前向后找偶数，找到后从当前位置向后找奇数，将奇数记在 temp，将偶数到奇数这段元素整体向后移一格，temp 赋给原偶数位置。
  • 不能将记偶数位置，记偶数位置相当于将后边的奇数向前压，若是出现相邻的偶数排列会出错，应选择将偶数向后压。

数值的整数次方

题目描述

给定一个double类型的浮点数base和int类型的整数exponent。求base的exponent次方。

思路

快速幂（要认识到任何一个整数都能用二进制表示，其中内涵很深）
若要求 3^20 ，按一般的方法，要进行 19 次乘法才能得到结果，但是可以注意到 3^20 = 9^10 = 81^5 ，那么只要 4 次运算就能得到结果。
现在求 A 的 156 次方，将 156 化为二进制 156（10）= 10011100（2），发现 A^156 = (A^4 ) (A^8 ) (A^16 ) (A^128 )。从二进制的右端算到左端，用位移符 >> 计算。

二进制中1的个数

题目描述

输入一个整数，输出该数二进制表示中1的个数。其中负数用补码表示。

思路

如果一个整数不为0，那么这个整数至少有一位是1。如果我们把这个整数减1，那么原来处在整数最右边的1就会变为0，原来在1后面的所有的0都会变成1(如果最右边的1后面还有0的话)。其余所有位将不会受到影响。
举个例子：一个二进制数1100，从右边数起第三位是处于最右边的一个1。减去1后，第三位变成0，它后面的两位0变成了1，而前面的1保持不变，因此得到的结果是1011.我们发现减1的结果是把最右边的一个1开始的所有位都取反了。这个时候如果我们再把原来的整数和减去1之后的结果做与运算，从原来整数最右边一个1那一位开始所有位都会变成0。如1100&1011=1000.也就是说，把一个整数减去1，再和原整数做与运算，会把该整数最右边一个1变成0.那么一个整数的二进制有多少个1，就可以进行多少次这样的操作。

变态跳台阶

题目描述

一只青蛙一次可以跳上1级台阶，也可以跳上2级……它也可以跳上n级。求该青蛙跳上一个n级的台阶总共有多少种跳法。

思路

f(1) = 1
f(2) = f(2-1) + f(2-2) //f(2-2) 表示2阶一次跳2阶的次数。
f(3) = f(3-1) + f(3-2) + f(3-3)
…
f(n) = f(n-1) + f(n-2) + f(n-3) + … + f(n-(n-1)) + f(n-n)

得出：f(n) = 2 * f(n-1)，(n>=2)

知识点

  • << : 左移运算符，num << 1，相当于 num 乘以2。

跳台阶

题目描述

一只青蛙一次可以跳上1级台阶，也可以跳上2级。求该青蛙跳上一个n级的台阶总共有多少种跳法。

思路

找规律可以总结出是斐波那契数列 f(n) = f(n-1) + f(n-2)，假设现在6个台阶，我们可以从第5跳一步到6，这样的话有多少种方案跳到5就有多少种方案跳到6，另外我们也可以从4跳两步跳到6，跳到4有多少种方案的话，就有多少种方案跳到6，其他的不能从3跳到6，所以最后就是f(6) = f(5) + f(4)。

旋转数组的最小数字

题目描述

把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。 输入一个非递减排序的数组的一个旋转，输出旋转数组的最小元素。 例如数组{3,4,5,1,2}为{1,2,3,4,5}的一个旋转，该数组的最小值为1。 NOTE：给出的所有元素都大于0，若数组大小为0，请返回0。

思路

有三种情况

  1. rotateArray [mid] > rotateArray [begin]
    最小的数肯定在rotateArray [mid] 的右边。
  2. rotateArray [mid] < rotateArray [begin]
    最小的数肯定在rotateArray [mid] 的左边。
  3. rotateArray [mid] == rotateArray [begin]
    这种情况不好判断，如 [1,0,1,1,1]，[1,1,1,0,1]，只有 begin++ 挨个判断

知识点

  • STL 中容器是否为空的判断条件 v.empty() 和 v.size()==0 的区别：对于std::vector来说，其实效率没区别，但是对于std::list来说(gcc 4.6-)，empty()的效率是O(1)，size()的效率是O(n)。
    C++11对size()的效率做了一致性的规定。所以在gcc 4.7+, std::list的复杂度也是O(1)了。原因是增加了一个“size”的变量来存长度。

用两个栈实现队列

题目描述

用两个栈来实现一个队列，完成队列的Push和Pop操作。 队列中的元素为int类型。

思路

入队：将元素进栈A
出队：判断栈B是否为空，如果为空，则将栈A中所有元素pop，并push进栈B，栈B出栈；
如果不为空，栈B直接出栈。
补充：(用两个队列实现栈的功能)
入栈：将元素进队列A
出栈：判断队列A中元素的个数是否为1，如果等于1，则出队列，否则将队列A中的元素 以此出队列并放入队列B，直到队列A中的元素留下一个，然后队列A出队列，再把 队列B中的元素出队列以此放入队列A中。

知识点

  • std::stack::empty
    true if the underlying container’s size is 0, false otherwise.
  • stack::pop 没有返回值
