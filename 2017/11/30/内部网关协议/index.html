内部网关协议OSPF

OSPF协议的基本特点

这个协议的名字是开放最短路径优先 OSPF (Open Shortest Path First)。 它是为了克服 RIP 的缺点在 1989 年开发出来的。
“开放”表明 OSPF 协议不是受某一家厂商控制,而是公开发表的.使用了 Dijkstra 提出的最短路径算法 SPF。
OSPF 只是一个协议名字,并不表示其他的路由选择协议不是”最短路径优先”。事实上，所有的自制系统内部使用的路由协议（包括RIP协议）都是要寻找一条最短路径。

OSPF 最主要的特征就是使用分布式的链路状态协议，而不是像 RIP 的距离向量协议，OSPF的三个要点和 RIP 都不一样：

  1. 向本自制系统中所有路由器发送信息。这里使用的是泛洪法（flooding）。
  2. 发送的信息就是与本路由器相邻的所有路由器的链路状态，但这只是路由器所知道的部分信息。
  3. 只有当路由信息发生变化时才向所有路由器用泛洪法发送信息。

OSPF 的更新过程收敛得很快尤其重点。

为了使 OSPF 能用于规模很大的网络，OSPF将一个自治区再划分为如干戈更小的范围，叫区域。

主干区域的标志符规定为 0.0.0.0，是用来连接其他在下层的区域。从其他区域来的信息都由区域边界路由器进行概括。主干区域的路由器叫主干路由器。

OSPF 不使用 UDP 而**直接用 IP 数据报传送。

OSPF的五种分组类型

  1. 问候分组;
  2. 数据库描述分组;
  3. 链路状态请求分组;
  4. 链路状态更新分组;
  5. 链路状态确认分组;

OSPF 规定，每两个相邻路由器每隔十秒要交换一次问候分组。这样就可以确认哪些路由器是可达的。
